# Decision Log — AI Trip Planner PRD

Canonical memory and audit trail. Every decision, change, and override recorded as the conversation unfolds.

## 2026-06-10

- **Intent:** Create. Fresh PRD, no prior in-progress run on this topic (existing prd-scuba-* runs are unrelated: Citizen Science MVP, Affiliate Link Integrity, Photo Quality).
- **Product:** Grounded AI trip-planner assistant for scubaseason.fun. Owns discover → plan → book journey without building a booking engine.
- **Scope decided with Josie (pre-PRD conversation):**
  - Leads with proprietary reef data (condition classification, seasonality, species, sites); supplements with LLM general travel knowledge (clearly labeled, not live).
  - Funnels to EXISTING affiliate operator/lodging links. Assistant is the funnel, not a booking engine.
  - Must run on FREE LLM tier, no credit card (Gemini free tier / Groq). AVOID live web-search grounding (cost/billing trigger). Model-only + own-data RAG.
  - Honest about gaps; never hallucinate prices/fares.
  - OUT OF SCOPE: real booking infra, live price/fare data, full-service trip ops.
- **Codebase grounding reality (verified 2026-06-10):**
  - 356 sites, 113 locations with populated bestMonths, conditionsByMonth, species, reef-health.
  - operators have `priceRangeUSD`; lodging PartnerLinks have `priceLevel` (1-4 budget→luxury). → affordability CAN be answered relatively.
  - `TripCostEstimate` type DEFINED but NO data file exists (scaffold only). → full trip-cost totals are a GAP. "Cheapest" answerable only via relative price tiers, not precise totals.
  - `getThereStructured` populated on only ~3 sites; plain-text `getThere` exists more widely. → travel-logistics grounding is thin; general-knowledge supplement carries most of it.
- **Decision (Josie, AskUserQuestion):** Affordability = "Tiers now, dollars later" — ship with relative Price Tiers (lodging priceLevel, operator priceRangeUSD), architect for an automatic upgrade to dollar ballparks once TripCostEstimate is populated. → FR-4 + FR-5.
- **Decision (Josie, AskUserQuestion):** Working mode = Fast path. Drafted full PRD with [ASSUMPTION] tags; iterate on review.
- **Drafted:** prd.md v1 — 7 features (FR-1..FR-11), Essential Spine + Cross-Cutting NFRs, Constraints/Guardrails (Safety/Privacy/Cost), Aesthetic/Tone, IA. Status: draft, pending Josie review.
