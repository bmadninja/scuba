5. **Dive-site sighting log + plain-labeled conditions** (depth / current / visibility / water temp — spelled out, no "vis").
6. `/method` consolidation + nav simplification (fold "Evidence gaps" into atlas "needs fresh eyes" sort).
7. Phase 3 polish that is NOT already done: atlas hero stat row, live last-sync card, card freshness line, card shadow + hover-lift.

### REJECT / CORRECT
- **Hyphenation fix** ("one-time write-up") — violates no-hyphens rule. Reword: e.g. "built on daily monitoring, not a single survey."
- **"Go two-column" framing** for location page — already done; do not rebuild a working layout.
- Any item marked ALREADY DONE above — audit/confirm only, do not re-implement.

### DEFER
- **Three.js rotating globe** — delight, not clarity; perf/maintenance tax; the brief is clarity. Ship cheaper static/2D first.
- **Gradient cards instead of photos** — contradicts the hero-photo authenticity work; keep real photography.

## Design principles carried into this run (from handoff, reconciled with house rules)
1. No repeated information — if it's in the stat strip, don't repeat it in the body.
2. Plain language — no "DHW", no "vis 20m", no "Open water+" jargon in diver copy.
3. Science-first, booking-second — left = data/conservation, right = trip planning (already the structure).
4. Thriving ≠ pristine — "near its natural baseline", never "perfect".
5. Fresh eyes framing — "hasn't been documented recently", never "no data".
6. **House rule: no hyphens in user-facing copy** (em dashes fine). Overrides handoff hyphenation.
